GetReparsePointInfo

главная | FarStandardFunctions
Функция FSF.GetReparsePointInfo позволяет для указанной символической связи получить информацию об источнике - путь к исходному каталогу/диску. 
int WINAPI GetReparsePointInfo(
  const char *Src,
  char *Dest,
  int DestSize
);

Параметры

Src
Источник данных. Строка должна содержать полное имя к символической связи и заканчиваться слешем ('\').
Dest
Строка назначения. Может быть равна NULL.
DestSize
Размер строки назначения Dest. Если Dest=NULL, то параметр DestSize игнорируется.

Возвращаемое значение

Реальный размер строки, размещенной в Dest или требуемый размер строки для случая, когда Dest=NULL.

0 - в случае ошибки:

  • функция не поддерживается (пользователь применил плагин, использующий функцию FSF.GetReparsePointInfo, не в Windows 2000)
  • неверный путь (выставляется системный номер ошибки ERROR_PATH_NOT_FOUND)
  • символическая связь находится на сетевом диске и в большинстве случаев случае реальное имя каталога, на который указывает связь, использовать невозможно.

Примечание

  1. Адреса Src и Dest могут совпадать.
  2. Функция работает только в Windows 2000 и выше.
  3. Возвращаемое функцией значение не несет никакой практической пользы для сетевых дисков.
  4. В отличии от ConvertNameToReal, функция FSF.GetReparsePointInfo применима только для симлинков.
  5. Некоторые замечания относительно понятия "Символическая связь" читайте здесь.
